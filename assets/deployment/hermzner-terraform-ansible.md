# Hermzner Terraform and Ansible Reference

## Terraform Configuration

### main.tf

**File**: `sources/hermzner/terraform/main.tf`

```hcl
terraform {
  required_providers {
    hcloud = {
      source  = "hetznercloud/hcloud"
      version = "~> 1.49"
    }
  }
}

# SSH key -- named ${var.server_name}-deployer to support state import
# when a server was manually destroyed but the key resource remains
resource "hcloud_ssh_key" "deployer" {
  name       = "${var.server_name}-deployer"
  public_key = var.ssh_public_key
}

# Optional firewall -- disabled by default
resource "hcloud_firewall" "provisioning" {
  count = var.cloud_firewall_enabled ? 1 : 0
  name  = "${var.server_name}-provisioning"

  rule {
    direction = "in"
    protocol  = "tcp"
    port      = "22"
    source_ips = [
      var.deployer_ip != "" ? var.deployer_ip : "0.0.0.0/0"
    ]
  }
}

resource "hcloud_server" "hermes" {
  name        = var.server_name
  server_type = var.server_type
  location    = var.location
  image       = var.os_type
  ssh_keys    = [hcloud_ssh_key.deployer.id]
}

# Attach firewall only when enabled
resource "hcloud_firewall_attachment" "provisioning" {
  count       = var.cloud_firewall_enabled ? 1 : 0
  firewall_id = hcloud_firewall.provisioning[0].id
  server_ids  = [hcloud_server.hermes.id]
}
```

### Precondition (Firewall)

In `main.tf`, a precondition on `hcloud_firewall.provisioning` enforces:

```hcl
lifecycle {
  precondition {
    condition     = var.cloud_firewall_enabled ? var.deployer_ip != "" : true
    error_message = "deployer_ip must be set when cloud_firewall_enabled is true."
  }
}
```

### Group Vars (all.yml)

**File**: `ansible/inventory/group_vars/all.yml`

```yaml
# --- Image ---
# Must be pinned to a digest. Use ALLOW_UNPINNED_IMAGE env var to override.
hermes_image_ref: ghcr.io/hermes-agent/hermes-agent:latest@sha256:abc123...

# --- Dashboard ---
hermes_dashboard_enabled: true
hermes_dashboard_port: 9119

# --- API ---
hermes_api_port: 8642

# --- Runtime ---
# Options: quadlet (default) or compose
hermes_runtime_backend: quadlet
# Auto-start containers after deploy
hermes_start_runtime: true

# --- Mnemosyne (SQLite-vec memory backend) ---
hermes_mnemosyne_enabled: false

# --- SSH Policy ---
# Options:
#   disabled_after_tailscale - SSH only on tailscale0 interface (default)
#   restricted               - SSH only from deployer IP
#   open_key_only            - SSH from anywhere with valid key
public_ssh_policy: disabled_after_tailscale

# --- SSH Hardening ---
sshd_hardening_enabled: true

# --- Backup ---
backup_encryption_enabled: false
backup_age_recipient: ""
backup_retention_days: 30

# --- CORS ---
hermes_cors_origins:
  - "http://localhost:9119"
  - "http://127.0.0.1:9119"
```

### Preflight Assertions

The playbook `site.yml` includes 8 preflight assertions before any role executes:

| Assertion | Condition | Error Message |
|-----------|-----------|---------------|
| Image pinned | `hermes_image_ref` contains `@sha256:` OR `ALLOW_UNPINNED_IMAGE` env var is set | "hermes_image_ref must be pinned to a digest" |
| CORS safe | No origin in `hermes_cors_origins` equals `*` | "Wildcard CORS origin is not allowed" |
| Runtime valid | `hermes_runtime_backend` is `quadlet` or `compose` | "Runtime backend must be quadlet or compose" |
| SSH policy valid | `public_ssh_policy` is one of 3 values | "Invalid SSH policy" |
| Bind mode safe | Bind mode is `localhost` | "Services must bind to localhost only" |
| Volume label | Label suffix is set | "Volume label required" |
| Backup encryption | If enabled, `backup_age_recipient` is set | "Age recipient required" |
| Deployer IP | If SSH policy is `restricted`, `deployer_ip` must be set | "Deployer IP required for restricted mode" |

## Container Security Reference

### Quadlet Template (hermes.container.j2)

**File**: `sources/hermzner/ansible/roles/hermes/templates/hermes.container.j2`

```ini
[Unit]
Description=Hermes Agent
After=network-online.target
Wants=network-online.target

[Container]
Image={{ hermes_image_ref }}
ContainerName=hermes
Pull=newer
RunInit=true
UserNS=keep-id
DropCapability=ALL
NoNewPrivileges=true
ReadOnly=true
Tmpfs=/tmp:size=512m,noexec,nosuid,nodev
Tmpfs=/run:size=64m,noexec,nosuid,nodev
Tmpfs=/.local:size=64m,noexec,nosuid,nodev
Environment=API_SERVER_ENABLED=true
Environment=API_SERVER_HOST=0.0.0.0
{% if hermes_dashboard_enabled %}
Environment=HERMES_DASHBOARD=1
{% endif %}
{% if hermes_mnemosyne_enabled %}
Environment=MNEMOSYNE_DATA_DIR=/opt/data/mnemosyne
{% endif %}
Environment=HERMES_CORS_ORIGINS={{ hermes_cors_origins | join(',') }}
EnvironmentFile=%h/.hermes/.env
Volume=%h/.hermes:/opt/data{% if container_volume_label %}:Z{% endif %}
PublishPort=127.0.0.1:{{ hermes_api_port }}:{{ hermes_api_port }}
{% if hermes_dashboard_enabled %}
PublishPort=127.0.0.1:{{ hermes_dashboard_port }}:{{ hermes_dashboard_port }}
{% endif %}
Exec=gateway run
PidsLimit=512
Memory=2g
CPUQuota=200%
Restart=on-failure
RestartSec=10s

[Install]
WantedBy=default.target
```

## Verification Assertions

### verify.yml (11 assertions)

1. **not_privileged**: `podman inspect hermes --format '{{.HostConfig.Privileged}}'` returns `false`
2. **userns_mode**: `podman inspect hermes --format '{{.HostConfig.UsernsMode}}'` contains `keep-id`
3. **cap_drop_all**: `podman inspect hermes --format '{{.HostConfig.CapDrop}}'` contains `ALL`
4. **no_new_privs**: `podman inspect hermes --format '{{.HostConfig.SecurityOpt}}'` contains `no-new-privileges:true`
5. **seccomp_not_disabled**: `podman inspect hermes --format '{{.HostConfig.SecurityOpt}}'` does NOT contain `seccomp=unconfined`
6. **apparmor_not_disabled**: `podman inspect hermes --format '{{.HostConfig.SecurityOpt}}'` does NOT contain `apparmor=unconfined`
7. **ports_localhost**: All published ports use `127.0.0.1:` prefix
8. **not_root**: `podman inspect hermes --format '{{.Config.User}}'` is non-empty and not `0` or `root`
9. **data_dir_perms**: `/home/hermes/.hermes/` has mode `0700` or stricter
10. **dotenv_perms**: `/home/hermes/.hermes/.env` has mode `0600` or stricter
11. **health_endpoint**: `localhost:8642/health` returns HTTP 200

## Post-Deploy Manual Commands

### Mnemosyne Provider Selection

```bash
# SSH into the server
ssh hermes@<tailscale-ip>

# List and select memory provider
podman exec -it hermes /opt/hermes/.venv/bin/hermes memory setup
# -> Select 'mnemosyne'

# Verify
podman exec -it hermes /opt/hermes/.venv/bin/hermes memory status
```

### Service Management

```bash
# View logs
journalctl --user -u hermes -n 100 -f

# Restart
systemctl --user restart hermes

# Stop
systemctl --user stop hermes

# Check container
podman ps -a
podman inspect hermes
podman logs hermes

# Run interactive command
podman exec -it hermes /opt/hermes/.venv/bin/hermes --help
```
